# 05. 의사결정 리포트 — 건기식·식품-약물 상호작용 엔진

> 2026-06-21 · 약사로 케어 PoC 종합 · 근거: `01~04` 산출물(전부 실호출 검증)
> 한 줄 결론: **`RxNorm(정규화) + MedData(판정) + PubMed/Claude(근거·해석)` 조합으로 MVP는 무료~$29/월에 기술적으로 가능. 단 진짜 난이도는 API가 아니라 ①한글→성분 사전 ②관련성 매칭 ③임상 신뢰성 게이트에 있다.**

---

## 1. 어떤 API 조합으로 가야 하는가

### 채택 (확정)
| API | 역할 | 비용 | 약관 | 검증 결과 |
|---|---|---|---|---|
| **RxNorm/RxNav** | 성분 정규화 | 무료, 키 불필요 | 공개·비독점, 출처표기 | ✅ 10/10 매핑(STEP 2) |
| **MedData** | 건기식 상호작용 1차 판정 | 무료 250콜/월 → $29/월 5천콜 | ⚠️ 무료티어 상업이용 회색 | ✅ 9/10 커버(STEP 3) |
| **PubMed (E-utilities)** | 근거 논문 보강 | 무료(기존 `/api/evidence`) | 공개 | 기존 백본 재사용 |
| **Claude API** | 기전 구조화·관련성 판정·문구 | 사용량 과금 | — | STEP 4 인터페이스 |

### 보류/제외
- **NIH ODS 직접**: 팩트시트 API가 **Cloudflare로 자동접근 차단(STEP 3)**. DSLD는 라벨 데이터만(상호작용 없음). → MedData가 ODS를 큐레이션해주므로 직접 연동 불필요. 근거 보강은 PubMed로 일원화.
- **NatMed / IMgateway**: 유료·라이선스 → 향후 업그레이드 카드(아래 §3).

### 권장 아키텍처 (STEP 4에서 동작 확인)
```
한글 건기식 + 약물 → [ko사전+RxNorm 정규화] → [MedData 판정]
   → [★관련성 매칭+dedup] → [PubMed 근거] → [Claude 해석] → 구조화 JSON(약사/환자용)
```

## 2. 한국 건기식 매핑의 현실적 난이도와 해결책

**핵심 통찰: "RxNorm 커버리지"는 문제가 아니다. "입력 정규화"가 진짜 일이다.**

- 10종 전부 RxNorm ingredient(IN)로 매핑 성공 → DB 자체는 충분.
- 그러나 ① **통칭→임상명 변환 필요**(비타민K→phytonadione, CoQ10→ubidecarenone), ② **개념 드리프트**(통칭 "omega-3"는 임상명과 다른 RxCUI로 풀림), ③ **프로바이오틱스는 통칭/속명 매칭 실패**(균주 단위 필요).

**해결책 (검증·구현됨):**
- **수동 정규화 사전(`ko_supplement_dictionary.json`)** — 통칭·별칭·OCR변형 → 영문명·정밀 RxCUI 고정. 시드 10종 완료. **확장이 핵심 자산.**
- 프로바이오틱스 등은 **대표균주 매핑 + 규칙(항생제 2시간 간격)** 으로 보완.
- 식품(자몽)은 supplement DB 밖 → **별도 소스(PubMed/규칙)**.

**난이도 평가: 中.** 사전 구축은 단순노동이지만 약사 도메인지식이 필요 → **솔로 약사-개발자에게 오히려 진입장벽이자 해자(moat).**

## 3. MedData 무료티어로 MVP 가능한가? 유료/NatMed 필수인가?

### 결론: **무료티어로 PoC·초기 MVP 가능. 사용자 증가 시 $29/월로 충분. NatMed는 당장 불필요.**

| 단계 | 티어 | 근거 |
|---|---|---|
| PoC/내부검증 | 무료 250콜 | 본 검증 전체가 22콜. 충분 |
| 초기 MVP(소수 사용자) | 무료~$29 | 캐싱 시 콜 급감(상호작용 쌍은 거의 불변) |
| 성장기 | $29~79 | 5천~2.5만 콜 |
| 정밀·법적방어 필요 시 | NatMed 등 | 근거등급·전문가 큐레이션 보강 |

### ⚠️ 최우선 리스크: 무료티어 상업 이용 회색지대
- 약관은 "Service 재판매/상업적 착취"만 금지하고 **티어별 상업 구분을 명시하지 않음**. 마케팅은 Starter($29)에만 "commercial use" 표기 → **무료=비상업 해석 여지**.
- **액션:** 상용 출시 전 ① **$29 Starter 결제**(가장 깔끔) 또는 ② **서면 확인 메일**. PoC엔 무방.

### 캐싱 전략(콜 한도 핵심)
상호작용 쌍은 거의 변하지 않음 → **(건기식, 약물클래스) 결과를 Supabase에 캐시**하면 동일 조합 재호출 0. 무료티어로도 상당 규모 커버 가능.

## 4. 데이터 신뢰성 리스크 (미국 기준 데이터를 한국 약사가 쓸 때)

| 리스크 | 내용 | 완화책 |
|---|---|---|
| **페어매칭 부재** | MedData는 건기식 전체 목록 반환, 약물쪽 자유텍스트·rxcui=null. 그대로 쓰면 노이즈/누락 | ✅ **관련성 매칭 레이어**(STEP 4 구현)로 해결 |
| **severity 과대평가** | Fish oil/warfarin "moderate"인데 최신 근거는 영향 미미. 3단계뿐·근거등급 없음 | PubMed 근거등급 병기, severity 절대시 금지 |
| **기전 단순화/오류** | CoQ10 "구조가 비타민K와 유사" 등 논쟁적 서술 | Claude 재서술 + 약사 검수 게이트 |
| **미국≠한국 제품** | 성분·함량·허가 상이, 한국 특화 건기식(홍삼 등) 미수록 가능 | ko 사전에 한국 특화 항목 보강, 미수록 시 "정보없음" 명시 |
| **규제(의료기기/의료행위)** | 상호작용 "판정"이 의료행위·SaMD 경계 | 기존 DUR과 동일하게 **정보제공+약사 상담 유도** 프레이밍, 단정 회피 |

**원칙:** API 결과를 그대로 신뢰하지 않는다. **정규화→관련성→근거→약사검수**의 다단 게이트가 신뢰성의 본질이며, 이것이 "약문약답·대기업이 안 하는 빈틈"을 메우는 차별점이다.

## 5. 다음 단계 — 그랜트(KOSMES) 확정 시 vs 미확정 시

### A. 미확정(현 상태) — 무자본 지속
1. **ko 사전 확장** — 한국 판매량 상위 건기식 50~100종 추가(약사 지식 활용, API 비용 0).
2. **캐싱 레이어** — Supabase에 상호작용 캐시 → 무료 250콜로 데모 가능 규모 확보.
3. ~~**PubMed 연동 마무리**~~ ✅ **완료** — `evidence.mjs`가 NCBI E-utilities로 실제 논문 반환(의존성 0). 오메가3×와파린·SJW×피임약 모두 정확한 근거 논문 적중, `confidence: evidence-backed`. (다음: Claude 한국어 요약 = 앱의 `/api/evidence` summarize 재사용)
4. **관련성 매처 고도화** — RxClass API(무료)로 `DRUG_HINTS` 자동화.
5. **데모 1종 완성** — "오메가3+와파린" 같은 대표 1케이스 환자/약사 화면까지 → 그랜트 피칭 자료.

### B. 확정 시 — 가속
1. **MedData Starter($29) 결제** → 상업이용 명확화 + 5천콜.
2. **Claude 해석 레이어 실연결**(interpret.mjs) → 약사/환자용 문구 자동생성.
3. **NatMed 등 유료 근거 평가** — 법적 방어·근거등급이 필요한 시점에 도입 검토.
4. **약사 검수 워크플로** — 상호작용 결과를 약사가 컨펌하는 B2B 루프(케어의 약국 CRM과 결합).
5. **규제 자문** — SaMD/의료행위 경계 법률 검토(기존 DUR shadow 정책 확장).

## 최종 권고
**진행 권장(GO).** 기술 리스크는 낮고(무료 API로 검증 완료), 차별점은 데이터가 아니라 **약사 도메인 기반 정규화·관련성·검수 파이프라인**에 있어 모방난이도가 높다. 무자본으로 ko 사전 확장 + 캐싱 + PubMed 연동까지 진행해 그랜트 결과와 무관하게 데모를 완성하고, 확정 시 $29 결제와 Claude 해석으로 즉시 가속한다. **유일한 선결 과제는 MedData 무료티어 상업이용 약관 확인.**
