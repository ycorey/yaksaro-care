# 06. 상호작용 엔진 완성도 평가 + 활용 방안

> 2026-06-29 · `05_decision_report.md`(GO 결정) 이후 실구현 상태 점검 + 재사용 전략
> 한 줄 결론: **프로덕션급 "근거 백본"은 완성, 진짜 차별점(정규화·매칭 게이트)은 시드/프로토타입. 엔진을 "상호작용"에 가두지 말고 5개 재사용 역량으로 분해해 내재화·B2B·성장에 동시 전개한다.**

---

## 1. 엔진의 구성 — 단일 모듈이 아니라 4+1 레이어

| 레이어 | 위치 | 역할 |
|---|---|---|
| **A. 근거 백본** | `src/lib/pubmed.ts` + `evidence-grade.ts` + `summarize.ts` + `app/api/evidence/route.ts` | PubMed 검색 → A/B/C 등급 → 한국어 요약 (2026-06-29 완성·프로덕션 배포) |
| **B. 정규화** | `src/lib/supplement-interaction/normalize.ts` (ko사전 + RxNorm) | 한글 건기식·약물 → 영문 성분/RxCUI |
| **C. 매칭 게이트** | `src/lib/supplement-interaction/relevance.ts` (`filterPairsForDrug`) | MedData 노이즈 목록 → 질의약물 관련 쌍만 선별 (★차별점) |
| **D. 판정·평가** | MedData(`meddata.ts`) + `interaction-poc/eval-harness/` | 상호작용 1차 판정 + 매처 precision/recall 회귀 채점 |
| (별개) DUR | `src/lib/dur.ts` + `interactions` 테이블(305k쌍) | 약물-약물 상호작용 (ETL 적재, 현재 shadow) |

## 2. 완성도 평가 (레이어별)

| 레이어 | 완성도 | 근거 |
|---|---|---|
| **A. 근거 백본** | 🟢 ~85% | 프로덕션 배포·등급 판정·30일 캐시·라이브 검증(A필터 {A:8}, metformin+B12 다기관RCT→A). 갭: Claude 해석이 상호작용 흐름에 미연결, 호출 비용 |
| **B. 정규화(ko사전)** | 🔴 ~15% | **시드 10종에서 멈춤.** "확장이 핵심 자산"이라 명시했으나 사실상 미착수 — 한국 사용자 입력 대부분이 `NORMALIZE_FAIL` 위험. **최대 병목** |
| **C. 매칭 게이트** | 🟠 ~30% | `DRUG_HINTS` 8약물 **손맞춤** → 미등록 약물엔 즉시 0점. rxclass/claude/hybrid는 **스켈레톤(미구현, exit 2)** |
| **D. 평가 하네스** | 🟢 ~70%(틀) / 🔴 ~10%(데이터) | 채점(TP/FP/FN·P/R/F1/F2)·회귀(`--compare`)·diff 프레임 우수. 그러나 골든셋 9케이스·rule 만점(1.000)은 **손맞춤 탓**이라 현재 변별력 0 |
| **앱 통합** | 🟠 shadow only | 027 캐시 + OCR 배선, 사용자 비노출. 규제상 올바른 단계화(shadow→약사모드→환자) |
| **임상 신뢰 게이트** | 🔴 설계만 | 약사 검수 워크플로 미구현. 골든 케이스 전부 `proposed`(검수 0), c6_p2 TODO 미확정 |

### 종합
- **건기식-상호작용 기능 자체 = 출시까지 ~35%.** 1케이스 데모는 동작하나 약물·건기식이 늘면 손맞춤이 무너진다.
- **근거 등급 서브엔진(A) = ~85%, 독립적으로 이미 쓸 수 있는 자산** (오늘 생활정보에 실제 적용).
- **강점(설계의 정직함):** eval 하네스 존재 · shadow 단계화 · "데이터가 아니라 파이프라인이 해자" 통찰.

### 출시를 막는 3대 병목 (우선순위)
1. **ko 정규화 사전 10→100+** — 없으면 입력 대부분 실패. 단순노동이나 약사 지식 필요 = 해자이기도.
2. **매칭 게이트 일반화** — rxclass(무료)로 `DRUG_HINTS` 자동화 → 골든셋 확장으로 변별력 증명.
3. **약사 검수 루프** — 규제 안전(정보제공+상담유도) + 모방난이도↑의 핵심.

## 3. 재사용 가능한 5개 역량

엔진을 "상호작용"에 가두지 말고 역량 단위로 본다.

1. **근거검색 + A/B/C 등급 + 한국어 요약** (백본 A) — 도메인 무관, 가장 성숙
2. **한글 → 정규(성분/RxCUI) 정규화** (B) — 검색·바코드·OCR과 공용 가능
3. **노이즈 목록 → 관련성 선별** (C) — 분류·필터 일반 패턴
4. **약사법 안전프레임 생성** (`safety-frame.ts`) — 규제 준수 콘텐츠 생성
5. **precision/recall 회귀 하네스** (eval-harness) — 분류기 교체 시 회귀 증명, 도메인 무관

### 가장 강한 3개 — 심층 코멘트
- **#3 DUR ⊕ 근거 융합 (가장 저평가):** 305k 약물-약물 쌍이 이미 있는데 shadow로만 묵힘. DUR=쌍, 백본=근거 등급. 합치면 '경고'가 아니라 '근거 등급 붙은 설명'. 백본 보유라 공수 낮음, 경쟁사 미개척 영역.
- **#2 약사 모드 근거 카드 (규제 최안전 첫 노출):** 환자 직접 판정=SaMD 경계, 약사에게 근거 카드 제시=정보제공. 약사가 검수자로 들어오며 매칭 게이트 미성숙·검수 루프(#병목3)를 동시에 완충.
- **#1 전역 근거 레이어 (최단 ROI):** evidence-grade는 도메인 무관 → OTC·약정보 페이지에 그대로 꽂으면 앱 전체가 일관된 "근거 신뢰" 톤. 공수 거의 0.

## 5. 추천 — 3묶음 동시 전개 + 다음 1수

- **B2B(수익):** #2 약사 모드 근거 카드 (규제 안전 첫 상용 surface + 검수 루프 확보)
- **성장(유입):** #4 블로그 근거 엔진 (오늘 SEO 작업과 직결)
