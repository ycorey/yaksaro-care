B2C — 환자 복약관리
무료
처방약 + 일반약(OTC) + 영양제를 한곳에서 기록하고, 끼니마다 챙기도록 돕는다.
1
📷 처방전 촬영
카메라/갤러리로 처방전을 찍으면 기기에서 먼저 압축(1600px) — 업로드 용량·413 오류 방지
/medications/ocrCanvas 압축
2
🔎 OCR 파이프라인 (자동)
CLOVA OCR로 글자 추출 → GPT가 약품·용법 파싱 → EDI 보험코드로 약 신원 교정 → 처방전 텍스트만 저장하고 원본 이미지는 즉시 파기
POST /api/ocrCLOVA+GPT-4o-miniuser_prescriptions🗑 이미지 파기
3
✅ 약 확인 · 저장
추출된 약을 확인·수정 후 저장(또는 처방약/일반약/영양제 직접 등록). 저장 시 약물 상호작용을 백그라운드로 점검(DUR shadow)
/medications/adduser_medicationsDUR shadow · 비차단
4
💊 약 지갑
처방전·일반약·영양제 묶음 보관. 약 이미지·성분명·만료 D-day 표시 (실버 세대용 큰 글씨)
/wallet
5
🕒 오늘 복약 체크
아침·점심·저녁·자기 전 4슬롯으로 복약 체크. 지연 알림, 전체 완료 시 축하 효과. 현재 상태와 이력을 함께 기록
/todaymedication_schedulesmedication_check_logs
6
📅 복약 캘린더
날짜별 복약 순응도를 한눈에(전부/일부/놓침). 체크 이력 로그를 집계해 표시
/calendarGET /api/calendar
7
🔔 복약 리마인더 (자동 발송)
끼니 시각마다 서버 cron 실행 → 푸시 구독 + 활성 복약 있음 + 아직 미체크 + 해당 알림 ON 인 사용자에게만 웹푸시
Vercel cron/api/cron/medication-reminderspush_subscriptions
＋
🔗 단골약국 연결 접점
약국 QR을 스캔하면 단골약국으로 자동 매핑. 미로그인 시 쿠키+리다이렉트로 로그인 후에도 매핑 보장
/store/[store_id]profiles.regular_pharmacy_id
＋
🤝 단골 약사에게 공개 동의 접점
설정에서 “단골 약사에게 내 약 목록 공개” opt-in. 언제든 끄면 약사 화면에서 즉시 사라짐
/settingsconsent_pharmacist_view
🧑‍⚕️
B2B — 약사/약국
유료 SaaS
동의한 단골 환자의 복약을 읽기 전용으로 보며 복약 상담에 참고한다.
1
🏥 약국 계정 발급
운영팀이 약국 계정을 수동 발급(약국 정보 + 약사 역할 부여). 셀프 가입은 중기 과제
pharmaciesrole = pharmacist운영자 수동
2
🔐 약사 로그인 · 역할 가드
약사 전용 로그인. /pharmacy/* 경로는 미들웨어·레이아웃에서 약사 역할만 통과(환자 접근 차단)
/pharmacy/loginproxy.ts role 가드
3
🔳 약국 QR 생성 · 인쇄
약국 식별자(store_id)를 셀프 발급하고 QR 코드 생성 → A4 인쇄. 카운터에 비치해 환자가 스캔
/pharmacy/qrstore_id (owner RLS)QR SVG · A4
↔
🔗 환자가 QR 스캔 → 단골 연결 접점
환자가 약국 QR을 스캔하면 그 환자의 단골약국으로 매핑됨 (← B2C 흐름의 ‘단골약국 연결’과 동일 지점)
/store/[store_id]
↔
🤝 환자의 공개 동의 대기 접점
단골로 연결돼 있어도, 환자가 설정에서 ‘공개’를 켜야만 약사 화면에 나타남 (← B2C ‘공개 동의’와 동일 지점)
consent_pharmacist_view = true
4
📋 단골 환자 대시보드
동의한 단골 환자 목록 + 환자별 활성 복약 종수. 모든 조회는 약사 본인 토큰 + RLS로만 — 동의 환자 것만 노출
/pharmacyuser token + RLS
5
👁 환자 복약 상세 (읽기 전용)
선택한 환자의 복약 목록을 read-only로 확인. 쓰기 경로 0 · 진단/처방변경/복약중단 지시 문구 0 — 참고 도구로 한정
/pharmacy/patients/[id]read-only
🔗두 영역이 만나는 곳 — 단골 연결과 동의
B2C와 B2B는 따로 동작하다가, QR 단골 연결과 환자의 열람 동의 두 지점에서만 이어진다. 약사가 환자 복약을 보려면 이 둘이 모두 성립해야 한다.

🛡 약사 모드 보안 계율
  • 관계(QR 단골) ≠ 동의 — opt-in 동의가 RLS의 필수 AND 조건
  • 약사 조회는 service_role 우회 없이 사용자 토큰 + RLS로만
  • 최소권한 — 약사 경로는 SELECT 전용, 환자 데이터 쓰기 경로 0
  • 처방전 원본 이미지는 텍스트 추출 즉시 파기(민감정보 비식별화)
  • DUR 상호작용은 백엔드 shadow 로깅 — 사용자 응답을 차단하지 않음
  • 의학적 판단(진단·처방변경·복약중단) 지시 문구 0 — 참고 도구로 한정
