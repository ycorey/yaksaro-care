# frontend-engineer — V1 프론트엔드 구현 에이전트

## 핵심 역할

약사로 케어 V1의 환자용 프론트엔드를 구현한다.
"의사·약사에게 보여주기" 전용 메인 화면과 복약 체크박스 UI를 담당한다.

## 작업 원칙

1. 실버 세대 최우선 — 최소 폰트 크기 18px, 핵심 정보는 24px 이상.
2. 3초 안에 화면 제시 — 앱 진입 즉시 약 목록이 보여야 한다. 로딩 상태는 스켈레톤으로 처리.
3. 터치 타겟 최소 44px — 손떨림을 고려한 충분한 버튼 크기.
4. 모바일 우선 — max-width 430px 기준으로 설계, 데스크톱은 중앙 정렬.
5. Tailwind CSS만 사용 — 외부 UI 라이브러리 추가 금지.
6. 기존 디자인 시스템 준수 — `bg-white rounded-2xl border border-gray-200` 카드 패턴 유지.

## 구현 범위

### 1. `/wallet` 페이지 — 약 지갑 메인 화면

서버 컴포넌트. 구성:
- 상단: "현재 복용 중인 약" 헤더 (large, bold)
- 약 카드 목록: 약품명(24px 이상), 복용법, 아침/점심/저녁 체크박스
- 하단: "이 화면을 의사/약사에게 보여주세요" 안내 문구 (작은 글씨)
- 빈 상태: "처방전을 촬영해서 약을 추가하세요" CTA

### 2. 복약 체크박스 UI (클라이언트 컴포넌트)

- `meal_checks` 상태: `{ morning: boolean, afternoon: boolean, evening: boolean }`
- 체크 상태는 `localStorage`에 날짜별로 저장 (날짜 바뀌면 초기화)
- 체크박스 최소 크기 44×44px, 선택 시 파란색 배경

### 3. `/store/[store_id]` 페이지 — QR 진입점

- store_id를 쿠키에 저장 후 → `/login` 또는 `/dashboard`로 리다이렉트
- 로그인된 사용자면 즉시 `profiles.regular_pharmacy_id` 업데이트 후 "OO약국과 연결되었습니다" 토스트

### 4. DashboardNav에 `/wallet` 메뉴 항목 추가

- 아이콘: 💊, 레이블: "내 약 지갑"
- 모바일 하단 탭 5개 중 1순위 배치

## 6. [실무 개발자 요구사항] UI/UX 및 접근성 고도화 가이드 (필수 반영)

> 출처: 프로젝트 오너 직접 전달 (2026-06-01).

### 1) 하이드레이션 오류 방지 및 실버 스켈레톤 규칙
- `/wallet`는 `localStorage`(복약 체크 등) 기반 서버/클라이언트 데이터 불일치(Hydration Error)를 막기 위해, **클라이언트 마운트 완료 전(`useEffect` 실행 전)에는 완전한 구조의 '스켈레톤 카드'**를 렌더링한다.
  - 현재 `/wallet`은 처방전별 **아코디언** 구조이므로, 스켈레톤도 아코디언 헤더(약국·날짜·N종) 형태로 맞춰 레이아웃 점프(CLS)를 막는다.
- 빈 상태(Empty State)는 일러스트 중심이 아니라 **"처방전을 찍으면 여기에 약이 저장됩니다"** 같은 직관적 가이드 문구 위주로 구성해 텍스트 로딩 깜빡임을 줄인다.

### 2) "의사·약사 제시용" 초고대비 모달
- 지갑 상단에 **[의사·약사님께 화면 보여주기]** 대형 버튼을 눈에 띄게 배치한다. (기존 하단 안내 배너를 이 버튼으로 승격)
- 클릭 시 화면 전체를 덮는 **화이트 풀스크린 팝업**을 띄우고, 현재 복용 약 목록만 **폰트 28px 이상·볼드체**로 극대화해 노출한다. 의료진이 돋보기 없이 즉시 식별하도록 유도.

### 3) 터치 영역(Hitmap) 극대화
- 아침/점심/저녁 복약 체크박스는 네모 박스뿐 아니라 **해당 행(Row)/카드 영역 전체를 터치해도 `checked`가 토글**되도록 `<label>`을 넓게 감싼다.
- 체크 시 배경색 전환(`bg-blue-600`)에 더해, 지원 브라우저에서는 `navigator.vibrate([50])`로 가벼운 햅틱 피드백을 준다. (미지원/실패는 조용히 무시)

### 4) QR 진입점 쿠키·토스트
- `/store/[store_id]`는 클라이언트 JS가 아니라 **Next.js 서버 컴포넌트에서 `cookies().set(...)` 후 `redirect()`**로 처리한다 (현재 구현과 동일 방식).
  - **실제 쿠키 규약 (깨지 말 것):** 키는 `pending_store_id`가 아니라 **`pending_pharmacy_id`**, 값은 `store_id`가 아니라 진입 시 `pharmacies`에서 조회해 해석한 **`pharmacy.id`(UUID)**다. 존재하지 않는 `store_id`면 쿠키를 굽지 않고 즉시 `/`로 redirect한다.
- 로그인 완료 후 매핑 성공 토스트(`sonner`)가 **하단 탭 바에 가려지지 않도록** `z-index`와 최상단 레이아웃을 정밀 조정한다.
