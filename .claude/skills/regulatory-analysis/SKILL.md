# 한국 의료 규제 분석 방법론

## SaMD 판정 기준 (의료기기법)

식약처 "소프트웨어 의료기기(SaMD) 가이드라인"에 따라 다음을 확인:

**의료기기 해당 가능성 높은 기능:**
- 특정 약물 복용 금지를 "처방"하는 기능
- 개인의 진단 결과를 기반으로 치료 방향을 제시하는 기능
- 임상 의사결정 지원 (Clinical Decision Support)

**의료기기 해당하지 않는 기능 (설계 방향):**
- 일반 정보 제공 ("이 두 약물은 상호작용이 있을 수 있습니다. 약사에게 확인하세요.")
- 복약 일정 알림
- 복약 이력 기록

**핵심 언어 설계 원칙:**
- 허용: "상호작용 가능성이 있습니다", "약사 또는 의사와 상담하세요"
- 금지: "이 약을 복용하면 안 됩니다", "처방을 중단하세요"

## 개인정보보호법 체크리스트

건강정보는 민감정보(제23조)에 해당한다. 다음을 필수 구현:

- [ ] 건강정보 수집 시 별도 동의 화면 (일반 개인정보 동의와 분리)
- [ ] 처방전 이미지 → 별도 저장 동의
- [ ] 개인정보처리방침에 민감정보 처리 목적·보존기간 명시
- [ ] DB 저장 시 건강정보 컬럼 암호화 (AES-256 이상)
- [ ] 처방전 이미지 Supabase Storage 암호화 저장
- [ ] 개인정보보호 책임자(CPO) 지정 및 공개

## 약사법 처방전 OCR 적법성

약사법 제23조 "처방전 재사용 금지"의 적용 범위 확인:
- OCR로 읽은 후 원본 처방전을 서버에 저장하는 행위가 "재사용"에 해당하는지
- 복약 관리 목적의 처방전 정보 디지털화 허용 여부
- 환자 본인 목적 보관은 일반적으로 허용되나, 약국 전달 목적은 별도 검토 필요
