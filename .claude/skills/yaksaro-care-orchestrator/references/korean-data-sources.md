# 한국 의약품·건강 공공 데이터 소스 레퍼런스

이 파일은 data-strategist 에이전트와 tech-architect 에이전트가 조사 시 참조하는 한국 공공 데이터 API 목록이다.
각 에이전트는 이 목록을 기반으로 WebSearch와 WebFetch로 최신 정보를 확인해야 한다.

---

## 1. 공공데이터포털 (data.go.kr)

통합 검색 경로: https://www.data.go.kr

### 식품의약품안전처 주요 API

| API명 | 키워드 검색어 | 데이터 유형 |
|------|------------|-----------|
| 의약품개요정보 | "의약품개요" OR "의약품허가정보" | 제품명, 성분, 효능 |
| 의약품낱알식별정보 | "낱알식별" | 약품 외형 (색상, 모양, 각인) |
| 의약품 주의사항 | "임부금기" OR "노인주의" | 특수 환자군 주의 |
| 의약품제조및수입실적 | "의약품 실적" | 유통량 데이터 |

### 건강보험심사평가원 주요 API

| API명 | 키워드 검색어 | 데이터 유형 |
|------|------------|-----------|
| DUR 품목정보 | "DUR" OR "병용금기" | 병용금기, 연령금기, 임부금기 |
| 의약품안전사용서비스 | "의약품 안전 심평원" | 약품 안전 정보 |
| 요양기관 현황 | "요양기관" OR "약국 현황" | 약국 수, 위치 |

**중요**: 심평원 DUR API는 공공데이터포털 외에 별도 협약이 필요할 수 있음.
공식 문의: https://www.hira.or.kr → 정보 → 개방데이터

### 식품안전나라 API

공식 포털: https://www.foodsafetykorea.go.kr

| API명 | 데이터 유형 |
|------|-----------|
| 건강기능식품 기능성 원료 정보 | 원료명, 기능성 내용, 일일 섭취량 |
| 건강기능식품 제품 정보 | 제품명, 제조사, 성분 |
| 식품첨가물 정보 | 첨가물명, 사용 기준 |

## 2. 약학정보원 e약은요

공식 사이트: https://www.e-drug.or.kr

제공 데이터:
- 낱알식별 서비스 (외형 기반 약품 검색)
- 의약품 정보 (효능, 부작용, 주의사항)
- 일반의약품 안전정보

데이터 접근 방법: 공공데이터포털 내 API 또는 별도 계약 확인 필요

## 3. 한국의약품안전관리원 (KIDS)

공식 사이트: https://www.kids.go.kr

제공 데이터:
- 의약품 이상사례 보고 통계
- 약물 이상반응 정보
- 복약안전 정보

API 제공 여부: 공공데이터포털 검색 키워드 "KIDS" OR "의약품 이상반응"

## 4. 해외 데이터 소스

### DrugBank
- 사이트: https://www.drugbank.com
- API: https://docs.drugbank.com
- 라이선스: 학술(Academic) 무료, 상업(Commercial) 유료
- 특징: 성분 기반 상호작용 데이터 가장 풍부
- 한국 약품: 국제 INN(성분명) 기반 → 한국 제네릭 커버리지 제한적

### RxNorm (NLM/NIH)
- API: https://rxnav.nlm.nih.gov
- 라이선스: 무료 공개
- 특징: 미국 FDA 승인 약품 표준 코드 체계
- 활용 방법: 성분명(INN) → RxNorm CUI → 상호작용 조회

### OpenFDA
- API: https://open.fda.gov/apis
- 라이선스: 무료 공개
- 제공 데이터: 미국 FDA 이상반응 보고(FAERS), 약품 라벨 정보
- 한국 약품: 미국 판매 약품만 포함

### Natural Medicines (Therapeutic Research Center)
- 사이트: https://naturalmedicines.therapeuticresearch.com
- 특징: 건강기능식품-약물 상호작용 DB. 업계 표준.
- 라이선스: 구독 기반 (조직 라이선스 수천~수만 달러/년)
- API 제공 여부: 별도 계약 필요
- MVP 적합도: 비용 부담 → V2 단계 검토 권장

## 5. 데이터 수집 대안 (단기)

DUR API 승인 대기 중인 M1 단계에서 사용할 수 있는 임시 대안:

1. **식약처 의약품 목록 정적 파일 활용**: 공공데이터포털에서 CSV/XML 다운로드 → Supabase에 직접 Import
2. **GoodRx/MedlinePlus WebSearch 보완**: 개발 테스트용 (프로덕션 사용 불가)
3. **약사 전문가 검수**: 초기 상호작용 규칙을 약사 파트너와 수동으로 구축 (핵심 50개 약물 페어)
