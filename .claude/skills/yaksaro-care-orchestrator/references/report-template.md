## 1. Executive Summary

**분량:** 1~1.5페이지 (핵심 결론만)

포함 항목:
- 서비스 한 줄 정의
- 핵심 발견 사항 3가지 (시장, 규제, 기술 각 1개)
- MVP 권고 범위 요약
- 투자 유치 가능성 한 줄 평가
- 가장 큰 리스크 1가지

---

## 2. 시장 검증 (Market Validation)

포함 항목:
- TAM / SAM / SOM (수치 + 근거)
- 핵심 고객 세그먼트 (나이대, 만성질환 유형, 다제복용 비율)
- 수요 검증 데이터 (부작용 건수, 다제복용 위험 통계)
- B2C 수익화 경로 (Freemium 전환 가정)
- B2B 약국 수익화 경로 (구독 모델, 약국당 LTV)
- 네트워크 효과 구조 다이어그램 (텍스트)
- 스위칭 코스트 분석
- 시장 진입 타이밍 평가

---

## 3. 데이터 전략 (Data Strategy)

포함 항목:
- 한국 공공 API 현황표
  | API명 | 제공기관 | 데이터범위 | 호출한도 | 비용 | 상업이용 | 업데이트주기 |
- DUR API 실제 활용 가능 범위
- 한국 공개 데이터로 커버 불가 영역
- 해외 데이터 보완 전략 (DrugBank/RxNorm/OpenFDA/Natural Medicines)
- 건기식 상호작용 데이터 확보 로드맵
- MVP 권장 데이터 스택

---

## 4. 규제 리스크 (Regulatory Risk)

포함 항목:
- 의료기기법 SaMD 판정 결과 및 근거
- 비의료기기 유지를 위한 기능 설계 기준
- 허용 표현 vs 금지 표현 가이드 (실제 UI 문구 예시)
- 개인정보보호법 필수 구현 체크리스트
- 약사법 처방전 OCR 적법성 판단
- 리스크 매트릭스 (심각도 × 발생가능성 × 완화방안)
- MVP 출시 전 필수 법적 조치 (우선순위 순)

---

## 5. 경쟁 분석 (Competitive Analysis)

포함 항목:
- 경쟁사 기능 비교표 (전체)
- 서비스별 페인포인트 (앱 리뷰 기반)
- 명확한 차별화 격차(Gap)
- 약사로 케어만의 USP (Unique Selling Proposition)
- 닥터나우·굿닥과의 보완재 vs 경쟁재 관계
- Medisafe 성공 요인 벤치마크
- 포지셔닝 맵 (텍스트)

---

## 6. 기술 설계 (Technical Design)

포함 항목:
- 기술 스택 최종 결정 및 이유
- ERD (텍스트 다이어그램)
- 핵심 DB 스키마 (CREATE TABLE SQL)
- RLS 정책 요약
- API 구조 (엔드포인트 목록)
- OCR 처리 플로우 (단계별)
- 약물 상호작용 분석 엔진 구조
- 보안 설계 (암호화, 인증, 처방전 이미지 보호)
- PharmMatch 재사용 컴포넌트 목록

---

## 7. MVP 정의 (MVP Definition)

포함 항목:
- MoSCoW 분류표 (기능별)
- 기능별 공수 추정 (일 단위, 낮음/중간/높음)
- 3개월 마일스톤 (M1/M2/M3)
- 외부 의존성 리드타임 및 선행 조치
- 기술 위험 Top 3 및 완화 방안
- 출시 성공 기준 (정량 지표)
- 솔로 개발 한계 및 아웃소싱 고려 영역

---

## 8. 향후 확장 로드맵 (Expansion Roadmap)

포함 항목:
- V1 → V2 → V3 기능 로드맵
- B2B 약국 SaaS 전환 타이밍 (언제 약국 구독 과금 시작)
- V2 건기식 상호작용 데이터 확보 전략
- V3 가족 관리 + 알림톡 확장
- 18개월 KPI 예측

---

## 9. 최종 투자 가치 평가 (Investment Value Assessment)

포함 항목:
- 한국 디지털 헬스 투자 환경 요약
- 비교 서비스 밸류에이션 표
- 약사로 케어 예상 밸류에이션 (시드/시리즈A 시나리오)
- 임팩트 투자 어필 포인트 (사회적 비용 절감 수치)
- 규제 리스크가 투자 유치에 미치는 영향
- 전략적 M&A 출구 시나리오
- Elevator Pitch 초안 (3문장)
- **최종 권고**: 부트스트래핑 vs 외부 투자 유치
